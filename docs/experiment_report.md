# 可视化智能语音交互控制系统实验报告

## 1. 实验目的

通过完成智能家居交互控制系统，掌握 GUI 界面开发、设备状态同步、语音信号处理、语音识别、手势识别、多线程并发、异常处理和模块化开发方法。

## 2. 实验环境

- 操作系统：Windows 11
- Python：3.10，Conda 环境 `gpt_smarthome`
- Java：OpenJDK 17
- 构建工具：Maven 3.9.15
- 后端框架：FastAPI
- 前端框架：JavaFX
- 语音处理：sounddevice、scipy、SpeechRecognition
- 手势识别：OpenCV、MediaPipe
- 语音合成：edge-tts

## 3. 实验内容完成情况

| 要求 | 完成情况 |
|---|---|
| 仿真界面 | JavaFX 房间模型、设备卡片、状态动画 |
| 语音频谱 | 后端 FFT 计算，前端实时折线图展示 |
| 滤波降噪 | scipy 带通滤波 |
| 语音控制设备 | 支持中文指令和 LLM 模糊理解 |
| 状态同步 | REST + WebSocket 实时同步 |
| 开始/停止监听 | 前端按钮调用后端接口 |
| 日志输出 | 前端日志框实时显示 |
| 异常处理 | 麦克风、摄像头、识别失败、资源缺失均有处理 |
| 手势识别 | 支持手掌、拳头、比耶、举手 |
| 声纹识别 | 支持录制样本和相似度校验 |
| 定时控制 | 支持延迟定时任务接口 |
| 界面美化 | 深色玻璃拟态、霓虹状态、动画效果 |
| 趣味音效 | 支持用户自定义设备音效 |
| 电视视频 | 支持用户放置 MP4 在虚拟电视中播放 |

## 4. 测试用例

| 编号 | 输入 | 预期结果 |
|---|---|---|
| T01 | 打开客厅灯 | 客厅灯状态开启，界面发光，播放 `light_on.mp3` |
| T02 | 关闭风扇 | 风扇状态关闭，旋转动画停止 |
| T03 | 打开电视 | 电视状态开启，播放 `tv_demo.mp4` |
| T04 | 进入观影模式 | 灯光调暗、窗帘关闭、电视开启 |
| T05 | 我要睡觉了 | 关闭电视和客厅灯，卧室灯低亮 |
| T06 | 太暗了 | 自动打开客厅灯 |
| T07 | 有点热 | 空调开启并调到 24 度 |
| T08 | 手掌手势 | 打开客厅灯 |

## 5. 问题与改进

- 在线语音识别依赖网络，可后续加入 Vosk、Whisper 等离线模型。
- MediaPipe 手势识别受光照影响，可增加摄像头预览和置信度显示。
- 声纹校验当前使用轻量特征，可后续替换为 ECAPA-TDNN 等专业模型。
- JavaFX 视频播放对编码格式敏感，建议统一转码为 H.264 + AAC。

## 6. 结论

系统完成了题目要求的核心功能，并在演示效果上增加了 LLM 管家、语音回复、自定义音效和电视视频播放。系统结构清晰，模块之间通过标准 Intent 和 WebSocket 事件解耦，便于继续扩展。
