# 可视化智能语音交互控制系统设计报告

## 1. 项目目标

本项目实现一套纯软件的智能家居交互控制系统。系统通过 JavaFX 3D 前端模拟真实家居设备状态，通过 Python 后端完成设备状态管理、语音识别、手势识别、LLM 指令理解、TTS 语音回复和 WebSocket 实时同步。

## 2. 总体架构

系统采用前后端分离架构：

```text
JavaFX 桌面前端
  - 3D 可视化仿真
  - 设备动画
  - 用户音效播放
  - 电视视频播放
  - AI 语音回复播放
        |
        | REST + WebSocket
        v
Python FastAPI 后端
  - 虚拟设备状态中心
  - 中文指令解析
  - LLM 意图解析
  - 麦克风录音 / 频谱 / 滤波 / 语音识别
  - 摄像头手势识别
  - 声纹校验
  - TTS 语音合成
```

## 3. 模块设计

### 3.1 设备状态中心

`DeviceStateManager` 统一维护虚拟设备状态，包括客厅灯、卧室灯、空调、窗帘、电视、风扇、加湿器、门锁和扫地机器人。所有控制入口最终都转换为标准 `Intent`，再由设备状态中心执行。

### 3.2 指令解析模块

系统内置关键词解析，支持固定中文指令和场景模式。若启用 LLM，后端会调用 OpenAI 兼容接口，将模糊语言解析为白名单 JSON 指令，避免模型输出直接控制程序。

### 3.3 语音模块

语音模块使用笔记本麦克风录音，经过带通滤波降低噪声，通过 FFT 计算频谱并推送给前端显示。语音识别默认使用 `SpeechRecognition` 的中文在线识别，也支持 `mock` 模式便于无麦克风演示。

### 3.4 手势模块

手势模块使用 OpenCV 打开摄像头，并用 MediaPipe Hands 提取手部关键点。根据手指伸展数量判断手掌、拳头、比耶和举手，对应控制灯光、电视和窗帘。后端会通过 WebSocket 实时推送当前摄像头预览画面、手势名称、手部边界框和 21 个关键点坐标，前端在手势识别预览区中显示摄像头画面，并用矩形框出识别区域、用线段连接手指关键点，便于演示时观察识别结果。

### 3.5 语音可视化模块

语音模块打开笔记本麦克风后，会以小块音频流方式采样。后端对每个音频块进行带通滤波，计算 RMS 音量、波形点和 FFT 频谱，并通过 WebSocket 推送给前端。前端实时显示麦克风状态、音量条、波形图和频谱图，满足语音信号可视化要求。

### 3.6 TTS 与音效模块

AI 管家的文字回复由后端 TTS 模块合成为音频，默认使用 `edge-tts`。设备开关音效由 JavaFX 前端播放，用户只需把自己的音频放入 `assets/sounds/` 并在 `config/media-map.json` 中配置。

### 3.7 电视视频模块

电视是虚拟设备。当前端收到电视开启状态后，会在电视屏幕区域播放 `assets/videos/tv_demo.mp4`。如果视频不存在，界面显示待机亮屏效果并记录日志。

### 3.8 3D 场景与 GPU 渲染

家居系统主视图已实现为 JavaFX 3D 场景。前端使用 `SubScene`、`PerspectiveCamera`、`Box`、`Cylinder`、`Sphere` 和 `PhongMaterial` 搭建客厅、卧室、玄关、门窗、电视、灯具等模型。设备状态仍由现有 WebSocket 事件驱动，前端把状态变化映射为 3D 材质、灯光强度、模型尺寸、位置和旋转动画。

九类虚拟设备均有 3D 表现：客厅灯和卧室灯通过球体灯泡与半透明光晕表现开关；窗帘通过模型宽度变化表现开合；电视通过 3D 屏幕亮灭表现状态；空调通过材质和位置细节表现运行；风扇通过 3D 扇叶旋转表现运行；加湿器通过雾气球体显隐表现工作；门锁通过颜色区分锁定和解锁；扫地机器人通过圆柱模型移动表现工作状态。

Windows 下 JavaFX Prism 会优先使用 Direct3D 图形管线，启动脚本已设置 `-Dprism.order=d3d -Dprism.vsync=true`。因此 3D 几何体、材质、透明度、动画和视频贴图可由 GPU 参与渲染。需要注意的是，MediaPipe Python 手势识别仍主要消耗 CPU；当前 OpenCV OpenCL 只用于摄像头帧缩放、翻转、颜色转换等预处理。若后续完整 3D 化，应把 3D 场景、设备卡片和摄像头预览分区渲染，避免摄像头 JPEG 解码和 3D 动画同时刷新导致 UI 掉帧。

## 4. 数据流

```text
用户语音/手势/按钮
  -> 后端识别或前端请求
  -> CommandProcessor 解析 Intent
  -> DeviceStateManager 更新状态
  -> EventHub WebSocket 推送
  -> JavaFX 刷新界面、播放音效和视频
  -> TTS 生成 AI 管家语音回复
```

## 5. 异常处理

- 麦克风不可用：后端捕获异常并推送日志。
- 摄像头不可用：手势模块停止并提示。
- 语音识别失败：跳过本轮，不影响 GUI。
- LLM 调用失败：自动回退到本地关键词解析。
- 音效或视频缺失：前端日志提示，不中断程序。
- 非法指令：返回 AI 管家提示语音。

## 6. 可扩展性

后续可扩展真实硬件适配器、MQTT 网关、更多场景模式、更精细的 3D 模型贴图、更多声纹样本和本地离线语音识别模型。
